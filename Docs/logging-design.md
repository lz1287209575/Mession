# Logging Design

这份文档定义 Mession 日志系统的当前目标。

## 当前目标

日志系统要优先满足三件事：

- 多服联调时能快速判断来源
- 主链路回归时能快速定位失败阶段
- 不引入过重外部依赖

## 当前原则

- 继续使用项目内自研 `MLogger`
- 日志优先服务于调试与验证，不先追求复杂功能
- 多服日志要能清楚区分来源

## 推荐方向

后续如果继续增强，优先级建议是：

1. 更好的模块与上下文信息
2. 更稳定的错误分类
3. 更适合脚本消费的关键状态输出

不建议当前优先去做：

- 很重的日志框架替换
- 复杂 sink 体系
- 过早的格式化能力扩展
