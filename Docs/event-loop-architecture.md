# Event Loop Architecture

Mession 当前采用主循环 + 子循环的结构。

## 当前模型

```text
MEventLoop
  -> MTaskEventLoop
  -> MNetEventLoop
```

含义是：

- 主循环负责驱动顺序
- 任务循环负责执行已投递任务
- 网络循环负责监听、连接、收包和断线

## 设计目标

这套结构的目标是：

- 保持服务器主线程模型清晰
- 把任务执行和网络 IO 解耦
- 让各服共享同一套运行框架

## 当前结论

对业务开发来说，最重要的不是事件循环内部细节，
而是理解：

- 各服都在统一模板下运行
- 网络事件和任务事件已经有明确边界
- 不需要为每个服务再自造一套循环模型
