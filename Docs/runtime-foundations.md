# ⚙️ Runtime Foundations

这份文档收敛记录 Mession 当前的基础运行时设施。

## 基础类型与公共能力

项目在 `Core/NetCore.h` 和 `Common/` 中统一封装常用能力，例如：

- 基础整数与字符串类型
- `TArray` / `TVector` / `TMap` / `TSet`
- 时间与唯一 ID
- `SVector` / `SRotator`
- 配置、字符串工具、结果类型

新代码建议优先使用项目别名，减少底层类型分裂。

## 事件循环

当前采用主循环 + 子循环结构：

```text
MEventLoop
  -> MTaskEventLoop
  -> MNetEventLoop
```

它的意义是：

- 任务执行和网络 IO 有明确边界
- 各服共用同一套运行模型
- 不需要每个服务重复发明生命周期框架

## 任务与异步

当前任务队列、线程池和异步设施主要用于：

- 后台任务执行
- 把任务投递到后续 tick
- 在调用链之间传递未来结果

原则是：

- 不把线程池变成业务状态机
- 优先服务当前事件循环模型
- 先求简单稳定，再求复杂调度

## 服务器模板

各服当前共享同一套运行模板，典型生命周期包括：

- 初始化配置
- 注册监听
- 驱动主循环
- 驱动网络和任务子循环
- Tick 后端连接
- 执行关服清理

新增服务时，优先沿用现有模板。
