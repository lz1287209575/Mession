顺着“去 Workflow”这条线怎么读

如果你的核心关注点还是“Workflow 已经被 Runtime 替掉了吗”，建议按下面顺序读源码。

第一步：看入口是否统一

从 Source/Servers/World/Player/PlayerService.h 开始，确认入口是不是统一收敛到了 PrepareRuntimeDispatch / DispatchRuntimeCommand 这一套契约。

第二步：看业务体是否顺序化

再看 PlayerEnter.cpp、PlayerSwitchScene.cpp、PlayerCombat.cpp，确认业务体是不是主要靠 MAwaitOk 顺序串起来，而不是回到 callback / workflow graph。

第三步：看 Runtime 是否真的兜底

最后看 PlayerCommandRuntime.h/.cpp、CommandExecutionContext.h、FiberScheduler.h，确认 strand、epoch、fiber continuation 有没有把复杂度吃掉。

第四步：看公共层是否继续抽象

如果未来还要继续做，就回头看 Servers/App 里的请求校验和 async glue 有没有继续沉淀为复用 helper。
