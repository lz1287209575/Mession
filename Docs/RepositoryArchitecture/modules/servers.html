业务层总览

这六个 server 的角色并不对称：World 是中心编排者，其它大多是能力型后端。

Gateway

客户端接入、转发到 World、接收下行推送。

Login

会话发放与校验。

World

玩家聚合、业务编排、client 入口和跨后端协调。

Scene

场景会话与战斗执行。

Router

玩家在线路由投影。

Mgo

玩家状态持久化。

分模块评价

这里保留最重要的设计判断，而不重复所有实现细节。

模块当前定位优点主要风险
Gateway接入边界足够薄，业务侵入少。未来若增加业务前置逻辑，会破坏无状态边界。
Login能力型 server职责清晰，维护简单。会话状态仍偏简单，生产能力有限。
World业务中心负责玩家聚合和跨后端编排，最近已收掉显式 Workflow。仍是复杂度最大汇聚点，最容易膨胀。
Scene执行域定位清楚，战斗 runtime 已单独切出。复杂技能和多场景分片会放大执行层复杂度。
Router投影层边界清楚，职责单一。后续若 route record 过胖，会反向承载业务真相。
Mgo持久化层把存储实现从 World 解耦出来。数据模型后续仍要继续细化。

World 的特别说明

World 是当前最值得继续精修的模块，所以单独强调它的设计原则。

当前正确方向

  • 客户端入口保持薄壳：WorldClient* 只做适配和回包。
  • 业务入口统一走 ValidateRequest → PrepareRuntimeDispatch → DispatchRuntimeCommand。
  • 真实逻辑写进 DoXxx(...) 顺序函数体。
  • 异步步骤统一用 MAwait / MAwaitOk。

接下来最该防的

  • 不要重新引入显式 Workflow / OnStep。
  • 不要把过多横切逻辑继续堆进 MPlayerService。
  • 不要让 Scene / Router / Mgo 的内部语义回流到 World 基础设施层。

MFuture<TResult<FPlayerSwitchSceneResponse, FAppError>> MPlayerService::PlayerSwitchScene(
    const FPlayerSwitchSceneRequest& Request)
{
    return DispatchRuntimeCommand<FPlayerSwitchSceneResponse>(
        Request,
        "PlayerSwitchScene",
        {
            EDependency::Scene,
            EDependency::Router,
        },
        &MPlayerService::DoPlayerSwitchScene);
}

TResult<FPlayerSwitchSceneResponse, FAppError> MPlayerService::DoPlayerSwitchScene(FPlayerSwitchSceneRequest Request)
{
    MPlayer* Player = FindPlayer(Request.PlayerId);
    if (!Player)
    {
        return MakePlayerSwitchSceneError("player_not_found", "PlayerSwitchScene");
    }

    const uint32 PreviousSceneId = Player->ResolveCurrentSceneId();
    if (PreviousSceneId != 0)
    {
        (void)MAwaitOk(LeaveSceneForPlayer(Request.PlayerId, PreviousSceneId));
    }

    const FSceneEnterResponse SceneResponse =
        MAwaitOk(EnterSceneForPlayer(Request.PlayerId, Request.SceneId));
    return TResult<FPlayerSwitchSceneResponse, FAppError>::Ok({Request.PlayerId, SceneResponse.SceneId});
}

TResult<FPlayerEnterWorldResponse, FAppError> MPlayerService::DoPlayerEnterWorld(FPlayerEnterWorldRequest Request)
{
    FLoginValidateSessionRequest ValidateRequest;
    ValidateRequest.PlayerId = Request.PlayerId;
    ValidateRequest.SessionKey = Request.SessionKey;

    const FLoginValidateSessionResponse ValidateResponse =
        MAwaitOk(WorldServer->GetLogin()->ValidateSession(ValidateRequest));

    FMgoLoadPlayerRequest LoadRequest;
    LoadRequest.PlayerId = Request.PlayerId;
    const FMgoLoadPlayerResponse LoadResponse =
        MAwaitOk(WorldServer->GetMgo()->LoadPlayer(LoadRequest));

    const FSceneEnterResponse SceneResponse =
        MAwaitOk(EnterSceneForPlayer(Request.PlayerId, Player->ResolveCurrentSceneId()));
    return TResult<FPlayerEnterWorldResponse, FAppError>::Ok(
        BuildPlayerOnlyResponse<FPlayerEnterWorldResponse>(Request.PlayerId));
}
