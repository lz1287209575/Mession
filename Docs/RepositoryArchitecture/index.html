Mession 仓库整体架构已经拆成按模块、按链路、按 review、按示例、按 runtime 查阅的文档目录

现在可以分别从模块设计、跨模块时序、架构优先级、真实接口示例和 World Runtime 设计五个入口进入，不需要再在一个超长 HTML 里来回滚动。

这次目录化后的结构

  • modules/：按模块看基础层与业务层。
  • flows/：按跨模块主链路看系统行为。
  • review/：按优先级和 review 视角看问题清单。
  • examples/：真实接口、运行示例、源码来源索引。
  • runtime/：World Runtime 设计说明。
  • assets/：共享样式资源。

推荐阅读顺序

  1. 先看模块导航，建立整体地图。
  2. 再看主链路页，理解系统怎么跑。
  3. 然后看示例页和源码来源索引，把抽象设计与真实文件对上。
  4. 最后看 World Runtime 设计页，确认“去 Workflow”后的长期路线。

当前总判断

这页保留最浓缩的结论，方便你快速确认当前设计重心。

优点

  • 仓库分层已经很清楚：底座、协议、桥接、业务 server。
  • World 的显式 Workflow 已基本收掉，业务主链路可读性明显提升。
  • App 层已经开始承担通用 async glue 的复用责任。
  • 文档现在也能从真实接口示例和源码索引反推整体设计，不再停留在概念层。

当前主矛盾

  • World 仍然是复杂度最大汇聚点。
  • 跨模块异步边界虽然已收敛，但仍需要持续保持公共化。
  • 后续重点应该是 Runtime 能力和边界收口，而不是回到流程对象设计。
