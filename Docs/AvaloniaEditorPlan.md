# Avalonia 编辑器落地方案

## 目标

把当前 `MObjectEditorService` 从“Web 原型 + 本地服务”演进为：

- `MObjectEditorService`：本地资产服务与导出编排
- `MObjectEditorAvalonia`：正式桌面编辑器

## 当前确定的技术路线

- 桌面 UI：Avalonia UI
- 语言：C# / .NET 8
- 后端复用：现有 `MObjectEditorService` HTTP 接口
- 数据流：
  - 编辑器修改行模型
  - 调用 `batch-save`
  - 需要时调用 `validate` / `export`

## 为什么不再继续加码 Web

- 用户期望是本地开发工具，而不是网页后台
- 大量数据编辑更适合桌面交互模型
- 后续会需要 `MObject` 通用属性编辑、引用跳转、批量操作、布局停靠
- 这些能力在 Avalonia 上更接近正式工具形态

## 当前第一阶段范围

1. 先把怪物配置迁到 Avalonia
2. 保留当前服务端接口，不动核心逻辑
3. 保证最小读写链路成立：
   - 读表
   - 改值
   - 批量保存
   - 校验
   - 导出/发布
