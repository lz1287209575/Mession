# EventLoop / Asio Survey

这份文档记录的是网络层演进的调研背景，而不是当前主线方案。

## 当前状态

当前主线仍然基于项目自有的事件循环与 socket 封装。  
Asio 相关内容当前只属于调研储备，不是正在执行的默认路线。

## 调研结论

主要比较过两类方向：

- 自研 Reactor / EventLoop
- Asio 风格异步模型

当前继续沿用自研模型的原因是：

- 现有主链路已稳定
- 当前瓶颈不在更换网络库
- 架构重点在协议、Gameplay、replication、persistence

## 什么时候再看这份文档

只有在下面情况之一出现时，才值得重新把它提到前面：

- 当前网络层成为明确瓶颈
- 需要跨平台线程模型重构
- 要求更强的异步 IO 能力
