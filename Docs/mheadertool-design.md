# MHeaderTool Design

`MHeaderTool` 是 Mession 当前自定义反射系统的代码生成工具。

## 目标

它的职责是把“声明”转成“运行时可用 glue”，减少手写注册代码。

当前主要服务于：

- 反射类与属性注册
- 函数注册
- 服务间 RPC 入口
- 客户端统一函数调用清单

## 当前结论

工具链的存在是主线，而不是样例。

也就是说：

- 新的反射类应优先靠生成链路接入
- 不应继续扩手写 `RegisterAllProperties` / `RegisterAllFunctions`
- 协议和函数入口应尽量从声明统一派生

## 当前短板

后续仍需继续补强：

- 更多声明形式覆盖
- unsupported 场景报错
- manifest 可靠性和可读性

## 当前建议

如果遇到反射或入口接入问题，先检查：

1. 声明是否符合工具链约定
2. 生成产物是否更新
3. 工具是否已经覆盖这种声明形式
